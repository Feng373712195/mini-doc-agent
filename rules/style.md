# 样式规范（必须严格遵守）

## 样式工程规范（必须严格遵守）

### 1. 全局样式（必须统一管理）

- 所有公共样式必须统一写入：
  /app/assets/app.css

- 禁止在任何组件 / 页面重复实现相同样式

---

### 2. 必须优先复用规则

在编写样式前必须：

1. 检查 app.css 是否已有相似样式
2. 若存在必须直接复用
3. 禁止重新定义相同视觉效果的样式

---

### 3. 公共样式定义规则

以下类型样式必须放入 app.css：

- layout（布局类）
- spacing（间距体系）
- typography（字体层级）
- card（卡片样式）
- button（按钮体系）
- input（表单样式）
- badge / tag（标签类）

---

### 4. 禁止行为（强约束）

- 禁止在组件中重复定义 card / button / layout 样式
- 禁止复制已有 class 逻辑
- 禁止“微调版重复样式”（只是改一点点也不允许）

---

### 5. Tailwind 使用规范

- Tailwind 是默认布局工具
- 仅用于页面结构和布局样式
- 禁止滥用 Tailwind 写复杂重复样式

---

### 6. 样式分层规则（非常重要）

#### 全局样式
- layout / theme / variables → app.css

#### 组件样式
- 组件独立样式 → 组件内部 <style lang="css">

#### 页面样式
- 页面级样式 → 页面文件内 scoped css

---

### 7. 禁止行为（强约束）

- 禁止在多个文件重复定义相同样式
- 禁止 inline style=""
- 禁止在 Tailwind 中写超长 class（超过 6 个组合需抽离）

## 样式违规处理机制

- 如果发现：
  - 使用 inline style

必须：
1. 停止当前任务
2. 重构样式结构
3. 再继续开发
