# 样式规范（必须严格遵守）

## 样式工程规范（必须严格遵守）

---

### 1. 全局样式（必须统一管理）

- 所有公共样式必须统一写入：
  /app/assets/styles/app.less

- 禁止在任何组件 / 页面重复实现相同样式

---

### 2. 必须优先复用规则（强约束）

在编写样式前必须：

1. 检查 app.less 是否已有相似样式
2. 若存在必须直接复用
3. 禁止重新定义相同视觉效果的样式
4. 禁止“轻微修改后重新实现”

---

### 3. 公共样式定义规则

以下类型样式必须放入 app.less：

- layout（布局类）
- spacing（间距体系）
- typography（字体层级）
- card（卡片样式）
- button（按钮体系）
- input（表单样式）
- badge / tag（标签类）

---

### 4. Less 使用规范（必须遵守）

- 必须使用 Less（.less）
- 必须使用变量管理设计系统（@primary-color 等）
- 必须使用嵌套结构组织样式
- 禁止使用纯 CSS（.css）
- 禁止在多个文件重复定义同一 UI 语义样式

---

### 5. 样式分层规则（非常重要）

#### 全局样式
- layout / theme / variables → app.less

#### 组件样式
- 组件独立样式 → 组件内部 <style lang="less">

#### 页面样式
- 页面级样式 → 页面文件内 scoped less

---

### 6. 禁止行为（强约束）

- 禁止在多个文件重复定义相同样式
- 禁止 inline style=""
- 禁止复制已有 class 逻辑
- 禁止微调后重复创建新样式
- 禁止滥用嵌套超过 3 层

---

### 7. 样式复用机制（核心规则）

- 所有 UI 必须优先使用 app.less 中已有样式
- 如果已有 card/button/layout 体系，必须直接复用
- 不允许“类似样式重新实现”
- 新样式必须说明无法复用原因

---

### 8. 组件样式规范

- 组件样式必须 scoped
- 组件样式不得覆盖全局样式
- 组件只允许做局部调整，不允许定义体系级样式

---

### 9. 样式违规处理机制（强制执行）

如果发现以下情况：

- inline style
- 重复定义已有样式
- 未使用 app.less 公共样式

必须：

1. 停止当前任务
2. 重构样式结构
3. 优先改为复用 app.less
4. 再继续开发
