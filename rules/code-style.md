# 代码风格规范

- ESLint + Prettier（必须）
- pnpm lint / format / build（提交前必须执行）
- 优先 Composition API
- 核心业务逻辑必须添加注释，说明“做什么 + 为什么这么做”
- 非直观的业务规则必须添加注释
- 公共函数（跨模块使用）必须使用 JSDoc 注释（包含参数和返回值说明）
- 简单函数或语义清晰的代码不强制添加注释，避免冗余
- 注释统一使用中文


## 优先只修改当前任务明确提及的文件。

- 如果实现任务必须修改关联文件：
    - 必须仅修改直接相关文件
    - 必须说明修改原因
    - 禁止扩散到无关模块

- 禁止：
    - 修改无关文件
    - 自动格式化整个项目
    - 自动修复其他模块
    - 回退未提及文件
    - 顺手重构
